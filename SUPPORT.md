# Getting Help

A quick map of where to ask what:

| You want to... | Go to |
|---|---|
| Report a bug in a specific plugin | The plugin's repo → `Issues` → `New issue` |
| Request a feature for a specific plugin | The plugin's repo → `Issues` → `New issue` |
| Report a security vulnerability | Follow [SECURITY.md](SECURITY.md) — *do not* file a public issue |
| Report a Code of Conduct violation | Email **github@oakensoul.com** |

## Before filing an issue

A little homework makes things move faster:

1. **Check existing issues** (open *and* closed) for the affected repo — there's a real chance someone hit it first.
2. **Check the repo's `CHANGELOG.md`** — your issue may already be fixed in a newer release.
3. **Try the latest version** of the plugin if you can. Many bugs are already-fixed-but-not-yet-installed.
4. **Reduce to a minimal reproduction.** "It doesn't work" is hard to debug; "running `/aida config` on a fresh project produces X but I expected Y" is easy.

## What we can't help with

- Help configuring Claude Code itself — start at [Anthropic's Claude Code docs](https://docs.claude.com/en/docs/claude-code).
- Help with proprietary or closed-source plugins not in this org.
- One-on-one consulting or paid support.

## Response times

This is an open-source project maintained on volunteer time. We aim to respond to active issues within a week, but there's no SLA. If something is genuinely time-sensitive, say so in the issue and we'll do our best.
